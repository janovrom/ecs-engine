## 2. Scope Delivered

### 2.1 Domain Grid and Terrain Editing
- `OccupancyGrid` supports 3D occupied/unoccupied cube cells.
- Lasso-style +1 elevation operation is implemented via selected-cell raise (`RaiseElevation`).

### 2.2 Movement, Climb, and Modifier Rules
- `MovementRules` implements:
  - diagonal cost = 1.5 for planar and vertical diagonals,
  - Sprint and Heavy Terrain configurable multipliers,
  - cancel-to-x1 rule when Sprint and Heavy Terrain both apply,
  - floor rounding for fractional movement budgets.
- `MovementPathfinder` provides budget-aware pathfinding over occupancy constraints with separate regular and climb movement budgets.
- Path validation enforces adjacency, occupancy checks, and movement budget compliance.

### 2.3 Preview and Commit Path Flow
- `PathPreviewSession` supports:
  - path preview generation (`TryBuildPreview`) from start/target,
  - validated commit (`TryCommitValidated`) with budget and occupancy checks,
  - tracked committed path and cost.
- `SimulationState` tracks final committed destination for end-to-end scenario assertions.

### 2.4 Scripted Operations
- Scripted operation model includes:
  - terrain operations (`setOccupied`, `raiseElevation`),
  - movement flow operations (`previewPath`, `commitPreviewPath`).
- JSON source-generation context updated for all operation types.

### 2.5 Tests
- Extended integration coverage for M7 in `M7DomainSliceTests`:
  - existing domain rule checks retained,
  - added pathfinding obstacle-routing test,
  - added validated-commit budget rejection test,
  - added end-to-end scripted scenario test (script load -> operations -> preview -> commit -> final state).
