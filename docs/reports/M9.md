## 2. Scope Delivered
- Observability contract defined: `ISystemExecutionObserver` with per-tick and per-system callbacks including batching-efficiency signals.
- `MetricsSystemExecutionObserver` implemented using `System.Diagnostics.Metrics` (OpenTelemetry-compatible) emitting 9 named instruments covering tick count, duration, allocation, batch count, batch size, batching efficiency, system run count, system duration, and system allocation.
- Severity-aware metric gating: `MinimumSeverity` on `RuntimeObservabilityConfig` gates per-system metrics at `Debug` level and tick-level metrics at `Information` and above; `Warning` or higher suppresses all current output.
- Configurable publish cadence: `PublishEveryTicks` controls sampling rate; disabled ticks incur ~2× overhead of a bare executor run.
- `RuntimeConfig.Observability` wired through JSON config loader (AOT-safe source generation), `RuntimeHostBuilder`, and `RuntimeHost.RunTick`.
- Pathfinder rewritten to budget-bounded A* with single movement pool, surface-walkable cell constraints, and climb as step-cost modifier (non-additive, aligned with D&D 5e movement model).
- Benchmarks added: `SystemExecutionBenchmarks`, `MovementPathfinderBenchmarks`, `QueryBenchmarks` — all in `tests/EcsEngine.Benchmarks` with `MemoryDiagnoser`.
- VS Code `build:benchmarks`, `run:benchmarks` tasks and `.NET Debug Benchmarks` launch config added.

## 5. Performance and Allocation Summary
Environment: Intel Core i5-8400 2.80GHz (Coffee Lake), Fedora Linux 43, .NET 10.0.8, X64 RyuJIT AVX2

### 5.1 Query Hot-Path (ECS QueryEach)
| EntityCount | Mean | Allocated |
|---|---|---|
| 100 | 225.590 ns | 120 B |
| 1 000 | 1,025.928 ns | 120 B |
| 10 000 | 12,095.055 ns | 120 B |

**Status: PASS** — Allocation is constant (120 B overhead per call, not per entity) across all entity counts. Scales linearly at ≈1.2 ns/entity.

### 5.2 System Execution (with and without Observability)
| SystemCount | No Observer | Metrics Sampled | Metrics Unsampled | Overhead (sampled) |
|---|---|---|---|---|
| 2 | 3.394 ns | 144.427 ns | 7.186 ns | 42× |
| 8 | 24.052 ns | 479.578 ns | 28.889 ns | 20× |
| 32 | 105.376 ns | 1,991.960 ns | 108.186 ns | 19× |

**Status: PASS** — Zero allocation in all executor cases. Disabled/unsampled path adds ~2–3 ns per system (3% overhead at 32 systems). Metrics observer adds ~58–60 ns per system when sampled; at `PublishEveryTicks > 1` the sampled overhead is amortised over N ticks. A 32-system full executor tick completes in 105 ns baseline; at 60 FPS budget of 16.7 ms this is well within the frame budget.

### 5.3 Pathfinder (Surface A*)
| GridSize | Open Grid Mean | With Obstacles (10%) Mean | With Obstacles (20%) Mean | Allocated |
|---|---|---|---|---|
| 16 | 124 µs | 251 µs | 234 µs | ~74 KB |
| 32 | 488 µs | 992 µs | 1,012 µs | ~184 KB |
| 64 | 2.1 ms | 5.0 ms | 4.9 ms | ~758 KB |

**Status: PASS** — Each individual pathfind call completes well under 1 second for realistic grid sizes (16–32). 64×64 open grid is 2.1 ms. A 64×64 grid with 10–20% obstacles is ~5 ms worst case. A D&D encounter map is typically 16×16 to 32×32 squares which falls in the 124–1,012 µs range.

**Notes:**
- Pathfinder uses managed allocations per call (priority queue + dictionary per search). Allocation is deterministic and scales with search space, not entity count.
- Hot-path allocation reduction (pooling frontier/dictionary) is a post-M9 optimisation candidate.

## 6. Observability/Logging Summary
- Enablement: Configurable via `RuntimeConfig.Observability.Enabled`. Default is `false`; must be explicitly opted in.
- Publish cadence: `PublishEveryTicks` (default 1, i.e. every tick). Values ≥ 2 skip sampling on non-modulo ticks.
- Severity: `RuntimeObservabilitySeverity` enum — `Debug` emits per-system metrics and tick metrics; `Information` emits tick metrics only; `Warning`/`Error` suppresses all current metric output.
- Release behavior: Fully operational in release; no runtime reflection. `System.Diagnostics.Metrics` is AOT-safe.
- Instruments emitted (meter name: `EcsEngine.Runtime`):
  - `ecs.tick.count` — counter (Information)
  - `ecs.tick.alloc.bytes` — counter (Information)
  - `ecs.tick.duration.ms` — histogram (Information)
  - `ecs.tick.batch.count` — counter (Information)
  - `ecs.tick.batch.max_size` — histogram (Information)
  - `ecs.tick.batch.efficiency` — histogram (Information)
  - `ecs.system.run.count` — counter, tagged by `system` (Debug)
  - `ecs.system.alloc.bytes` — counter, tagged by `system` (Debug)
  - `ecs.system.duration.ms` — histogram, tagged by `system` (Debug)
