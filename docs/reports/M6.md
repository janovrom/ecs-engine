## 1. Executive Summary

## 2. Scope Delivered

### 2.1 Runtime Host and Config
- Added `StartupMode` enum with `Local` and `Server` modes.
- Added `RuntimeConfig` and `RuntimeConfigLoader` for JSON startup config loading.
- Added `RuntimeHostBuilder` with:
  - explicit registration window (registration closes after `Build()`),
  - deterministic runtime registration ordering by system `Type.FullName`,
  - startup mode selection,
  - server-mode transport requirement enforcement.
- Added `RuntimeHost` with composed runtime state (`StartupMode`, services, optional transport, scheduler executor) and `RunTick(EcsWorld)`.
- Updated runtime entrypoint to boot from config file and startup mode.

### 2.2 Constructor-Only DI (Runtime)
- Added `RuntimeServiceCollection` and `RuntimeServiceProvider`.
- Runtime systems are registered via explicit constructor factories (`Func<RuntimeServiceProvider, TSystem>`), enforcing constructor-only dependency flow with no reflection-based activation.

### 2.3 Transport
- Replaced placeholder transport project content with:
  - `ITransport` abstraction,
  - `TransportMessage` envelope,
  - `InMemoryTransport` thread-safe implementation (cross-thread queueing, monotonic sequence IDs).

### 2.4 Tests
- Added `RuntimeHostIntegrationTests` covering:
  - startup mode config parsing,
  - server mode transport requirement,
  - runtime registration window closure,
  - deterministic runtime registration order,
  - constructor injection behavior,
  - in-memory transport cross-thread publish/consume behavior.
- Updated integration test project references to include runtime and transport projects.
