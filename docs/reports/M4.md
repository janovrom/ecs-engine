## 1. Executive Summary
- Objective: Introduce a fluent dependency-declaration API for systems, compile-time graph validation via Kahn's topological sort, and a single sequential execution entry point (`SystemExecutor.Run`).
- Outcome: Completed.
- Decision: Ready for M5.

## 2. Scope Delivered
- Scheduling contracts (`src/EcsEngine.Core/`):
  - `EcsSystemAttribute.cs` — `[EcsSystem]` marker attribute; reserved for Roslyn source generation in M10; runtime registration requires `IEcsSystem`.
  - `IEcsSystem.cs` — `static virtual void Configure(ISystemBuilder builder) { }` (default no-op) + `void Execute(EcsWorld world)`. Static dispatch via generic constraint (`T.Configure(builder)`) is AOT-safe; no reflection.
  - `SystemSchedulingException.cs` — thrown on cycle or unresolved-reference; exposes `IReadOnlyList<Type> InvolvedTypes`.
- Scheduling implementation (`src/EcsEngine.Core/Scheduling/`):
  - `ISystemBuilder.cs` — fluent public interface: `ReadComponent<T>()`, `WriteComponent<T>()`, `RejectComponent<T>()`, `After<TSystem>()`, `Before<TSystem>()`, `After(Type)`, `Before(Type)`.
  - `SystemBuilder.cs` — internal concrete implementation, collects sets into five `HashSet<Type>` properties.
  - `SystemScheduler.cs` — `Register<T>(T system)` invokes `T.Configure(builder)` via static dispatch; `Build()` validates constraint references, builds directed edge graph, runs Kahn's algorithm with a `SortedSet<string>` on `FullName` for deterministic tie-breaking (D-052), detects cycles via leftover in-degree.
  - `SystemExecutor.cs` — holds the sorted `IEcsSystem[]`; `Run(EcsWorld world)` iterates sequentially (D-033, D-051); exposes `IReadOnlyList<IEcsSystem> Systems` for introspection.
- Tests:
  - `tests/EcsEngine.Core.Tests/SystemSchedulerTests.cs` — 12 tests.

### 4.4 AOT/Reflection Validation
- `T.Configure(builder)` is dispatched statically through the generic constraint on `Register<T>` — no reflection, fully AOT-safe.
- `[EcsSystem]` attribute is not read at runtime in M4; it is a no-op until M10 Roslyn source generation.
- `SystemScheduler.Build()` uses `Type.FullName` string comparison only, which is safe under trim/AOT.

## 5. Performance and Allocation Summary
- `SystemExecutor.Run` iterates a pre-built `IEcsSystem[]` with a single `foreach` — no allocation per call.
- `SystemScheduler.Build()` is a one-time setup operation; collections (`Dictionary`, `HashSet`, `SortedSet`) are allocated only during graph construction.
- No benchmarks written for M4; scheduler overhead is structurally O(N) in system count and not on any hot path.
